# Roadmap — outcome-based phases

> Phases are defined by the outcome that becomes true, not by feature lists.
> Advancing a phase requires the prior phase's exit evidence. Dates are deliberately
> absent until Phase 1 velocity data exists.

## Phase 0 — Foundation *(current; authored — pending product-owner ratification)*

**Outcome:** every future addition is coherent, measurable, explainable, and hard to
derail. **Exit evidence:** ratified constitution/invariants/decisions; valid contracts;
specified slice; runnable contract validation; FOUNDATION_REPORT accepted.

## Phase 1 — Truth works

**Outcome:** the Brand Context Package pipeline runs end-to-end for both input modes on
real and benchmark inputs — sources classified, claims extracted with provenance,
contradictions surfaced, assumptions ledgered, gaps listed — with run records written
by construction (INV-OBS-001) and G4 deterministic checks green.
**Exit evidence:** EXP-0001 passes; a BC-001-style archive and 3 prompt-only briefs
produce valid brand-context packages within budget; zero unsupported claims escape the
validator on seeded tests.

## Phase 2 — Direction works

**Outcome:** three genuinely different territories per brief, diversity measured
(EXP-0002), selection + preference memory persisted, direction package (DNA, visual
world, design-system compile) valid and consumed downstream.
**Exit evidence:** EXP-0002/0003 pass criteria; blind reviewer distinguishes NABCor
territory sets from single-idea-three-colorways sets.

## Phase 3 — The slice closes

**Outcome:** homepage spec + three connected social specs generated from the direction
package; claim-bound copy; G1–G4 gates wired for spec-level checks; evaluation report
with reasons; the full slice runs on the benchmark set within budget.
**Exit evidence:** slice acceptance criteria in `docs/FIRST_VERTICAL_SLICE.md` §10 all
green; EXP-0004/0005 complete.

## Phase 4 — Production bridge

**Outcome:** website-spec compiles through the existing nabcor channel layer to a
rendered, gate-passing site (G1/G2/G3 on real renders); deployment-readiness (G5)
implemented; first real client delivered through the full spine.
**Exit evidence:** BC-002 registered — delivery measured by construction, compared to
BC-001 anchors and the ≥50% delivery-time target (`docs/delivery-baseline.md`).

## Phase 5 — Memory compounds

**Outcome:** second project for a returning brand starts from verified truth and
recorded preferences; learnings demonstrably alter behavior (quirk library consulted,
rejected directions not re-proposed).
**Exit evidence:** measured context/token delta and zero re-litigated decisions on the
returning-brand run.

## Later (each requires its own decision record — see NON_GOALS deferrals)

Additional channels (presentations, ads, video concepts) · performance-outcome
ingestion · preference learning across brands · publishing automation ·
generation-at-scale infrastructure.
